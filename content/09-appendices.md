## Appendix A — The online Handbook

The most important deliverable of this thesis is not contained in this
PDF. It is the **AUCOOP Community Network Handbook**, an online,
versioned, openly-licensed handbook hosted on GitHub and continuously
maintained by the AUCOOP association.

### A.1 Where to find it

- **Web (canonical, evolving):** the latest version of the handbook is
  published as a static website built from the `main` branch of the
  source repository. The site is the recommended entry point for
  readers who want to browse, search, or follow internal cross-links.
- **PDF (offline, expedition copy):** a self-contained PDF is published
  as a release artefact alongside each tagged version of the handbook,
  intended for download before travelling to a site with limited or
  unreliable connectivity (see §3.C.4).
- **Source repository:** <https://github.com/aucoop/Community-Network-Handbook>
- **Thesis-pinned snapshot:** the handbook content described in §3
  corresponds to branch `dev_mj_thesis` at commit
  `a5fc80b26368fe6cf1475994a12cc8f1967b84f0`, which collects the four
  feature branches merged for the purpose of this thesis (see Appendix
  E).

### A.2 Structure recap

The handbook is organised in four top-level chapters with a strict
1-to-1 mapping between Chapter 2 (the imaginary use-case story) and
Chapter 3 (the recipe guide):

- **Chapter 1 — Introduction.** Editorial purpose, audience, and how to
  read the handbook.
- **Chapter 2 — Imaginary Use Case.** A fictional community network
  told as a story, with one section per challenge, in narrative
  second-person voice with italic question titles.
- **Chapter 3 — Guide.** Step-by-step recipes, one folder per
  technology (`Network-Planning`, `IP-Addressing`, `Wireless-Mesh`,
  `Antennas`, `Power-and-UPS`, `Laptop-Deployment`, …).
- **Chapter 4 — Real Use Cases.** Concrete deployments used as case
  studies (Namibia 2026 is the first such entry).

The structural rules and information architecture are described in
detail in §3.C.2. Rule files governing voice, structure, and
navigation conventions are reproduced in Appendix G.

### A.3 Contributing

The handbook accepts contributions from anyone via the standard GitHub
pull-request workflow. New contributors are expected to read the
contributing guide and the four `.opencode/rules/*.md` files (see
Appendix G) before editing. The lightweight governance contract is
described in §3.C.6.

### A.4 How to cite the handbook

When citing the handbook, please cite the **specific commit or release
tag** consulted, not the moving `main` branch, so that the reference
remains reproducible:

> AUCOOP. *Community Network Handbook*. Version `dev_mj_thesis @
> a5fc80b`. <https://github.com/aucoop/Community-Network-Handbook>.
> Accessed YYYY-MM-DD.

The handbook should also be cited in conjunction with this thesis and
its companion [Motje, 2026], since the three artefacts together
constitute the full record of the project.
